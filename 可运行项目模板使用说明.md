## 🚀 在此基础上继续开发的注意事项

### 第一步：启动前必做的修改

```toml
# pyproject.toml — 改项目名和作者
[project]
name = "你的新项目名"
description = "项目描述"
authors = [{name = "你的名字", email = "your@email.com"}]
```

```bash
# .env — 以下三项必须重新配置，不能沿用旧值！
DATABASE_URL=postgresql+asyncpg://user:pass@localhost:5432/新数据库名
SECRET_KEY=重新生成一个强密钥（openssl rand -hex 32）
REDIS_URL=redis://localhost:6379/0
```

```bash
# 重新初始化 alembic 迁移历史（不要用旧的 versions/）
conda activate fastapi_env ; uv run alembic revision --autogenerate -m "init"
conda activate fastapi_env ; uv run alembic upgrade head
```

---

### 第二步：`api_router.py` 的处理

**复制后立即修改**，删除当前项目特有的路由注册：

```python
# 删除这行（admin 是当前项目特有的）
from app.domains.admin.router import router as admin_router
api_router.include_router(admin_router, prefix="/admin", tags=["admin"])

# 保留这两个（通用）
api_router.include_router(auth_router, prefix="/auth", tags=["auth"])
api_router.include_router(users_router, prefix="/users", tags=["users"])
```

---

### 第三步：`.importlinter` 按新项目调整

```ini
# 修改 domains_independent 中的模块列表，改成你新项目的领域名
[importlinter:contract:domains_independent]
modules =
    app.domains.users
    app.domains.你的新领域1    ← 替换
    app.domains.你的新领域2    ← 替换
```

---

### 第四步：开发新业务领域的标准流程

每增加一个业务模块（如 `products`），按此顺序操作：

```
1. 在 app/db/models/ 新建 ORM 模型文件
2. 在 app/domains/新领域/ 新建以下文件：
   ├── __init__.py
   ├── constants.py    ← 错误码 + 成功文案
   ├── schemas.py      ← Pydantic 输入/输出 Schema
   ├── repository.py   ← 继承 BaseRepository[Model] 
   ├── service.py      ← 业务逻辑，注入 Repository
   ├── dependencies.py ← DI 组装
   └── router.py       ← APIRouter 路由定义
3. 在 app/api_router.py 注册新路由
4. 运行 alembic 生成迁移
```

### ⚠️ 关键约束提醒（来自 `fastapi-standard.md`）

| 约束 | 说明 |
|---|---|
| **域间禁止横向调用** | `domains.A` 不能 import `domains.B`，跨域编排写在 `services/` |
| **Service 禁止直接用 ORM 模型** | Service 只操作 Schema，通过 Repository 隔离 ORM |
| **错误码必须定义在 `constants.py`** | 禁止在 router/service 中硬编码错误信息 |
| **所有异步操作必须用 `await`** | 禁止在 async 函数中使用同步阻塞调用 |
| **`fastapi-standard.md` 是唯一规范** | 开发前务必让 AI 先读取此文件 |
